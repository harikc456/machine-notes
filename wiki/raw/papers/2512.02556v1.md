# Conditional Memory via Scalable Lookup: A New Axis of Sparsity for Large Language Models

**Authors:** Xin Cheng, Wangding Zeng, Damai Dai, Qinyu Chen, Bingxuan Wang, Zhenda Xie, Kezhao Huang, Xingkai Yu, Zhewen Hao, Yukun Li, Han Zhang, Huishuai Zhang, Dongyan Zhao, Wenfeng Liang  
**Affiliation:** Peking University, DeepSeek-AI  
**Venue:** arXiv preprint, January 2026  
**arXiv:** 2512.02556

## Summary

Proposes **conditional memory** as a complementary axis of sparsity to MoE. While MoE uses conditional computation (dynamic routing to experts), conditional memory uses static lookup (indexing into an N-gram embedding table). The instantiation is **Engram**, a modernized N-gram module that enables O(1) lookups for fixed, stereotyped knowledge patterns.

## Motivation

Language modeling involves two qualitatively different tasks:
1. **Compositional reasoning:** Requires deep computation (MoE handles this)
2. **Knowledge retrieval:** Often static and local — common multi-token entities, formulaic phrases, stereotyped patterns. Standard Transformers simulate this expensively via early-layer attention + FFN reconstruction.

Engram delegates local static patterns to a lookup table, freeing backbone layers for reasoning.

## Engram Architecture

- Modernized N-gram: local context indexes a massive embedding table via constant-time O(1) hash lookups
- Modern adaptations: tokenizer compression, multi-head hashing, contextualized gating, multi-branch integration
- Deterministic addressing enables runtime prefetching (overlaps communication with computation)
- 100B-parameter table offloaded to host memory with <3% overhead

## Sparsity Allocation Law

Formalizes the **Sparsity Allocation** problem: given a fixed total parameter budget, how to split between MoE experts and Engram memory? Experiments reveal a U-shaped scaling law — optimal performance uses a mix of both primitives, with neither extreme being best.

## Results (Engram-27B)

- Beats iso-parameter and iso-FLOPs MoE baseline across diverse domains
- Knowledge retrieval tasks: MMLU +3.4; CMMLU +4.0; MMLU-Pro +1.8
- General reasoning: BBH +5.0; ARC-Challenge +3.7; DROP +3.3
- Code/math: HumanEval +3.0; MATH +2.4; GSM8K +2.2
- Long-context: Multi-Query NIAH 84.2 → 97.0; Variable Tracking 77.0 → 89.0

## Mechanism

LogitLens analysis shows Engram relieves early layers from static knowledge reconstruction, effectively increasing network depth available for complex reasoning.

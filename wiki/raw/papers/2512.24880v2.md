# mHC: Manifold-Constrained Hyper-Connections

**Authors:** Zhenda Xie, Yixuan Wei, Huanqi Cao, Chenggang Zhao, Chengqi Deng, Jiashi Li, Damai Dai, Huazuo Gao, Jiang Chang, Kuai Yu, Liang Zhao, Shangyan Zhou, Zhean Xu, Zhengyan Zhang, Wangding Zeng, Shengding Hu, Yuqing Wang, Jingyang Yuan, Lean Wang, Wenfeng Liang  
**Affiliation:** DeepSeek-AI  
**Venue:** arXiv preprint, January 2026  
**arXiv:** 2512.24880

## Summary

Proposes Manifold-Constrained Hyper-Connections (mHC), a framework that projects HC residual matrices onto the Birkhoff polytope (the set of doubly stochastic matrices) to restore the identity mapping property lost in unconstrained HC. Uses iterative Sinkhorn-Knopp (SK) normalization for the projection.

## Problem with Hyper-Connections (HC)

Hyper-Connections generalize residual connections with dynamic mixing matrices, but unconstrained HC:
- Loses the identity mapping property intrinsic to residual connections
- Causes training instability (gradients can explode when non-identity matrices compose across layers)
- Incurs memory access overhead from large residual matrices

## mHC Solution

- Projects residual matrices onto the Birkhoff polytope (doubly stochastic matrices: all rows and columns sum to 1)
- Doubly stochastic matrices preserve spectral norm ≤ 1; closed under multiplication → prevents gradient explosion across layers
- Projection achieved via 20 iterations of Sinkhorn-Knopp normalization
- Infrastructure: kernel fusion, recomputing (gradient checkpointing), overlapping communication in DualPipe

## Results

- Effective at scale; tangible performance improvements over HC
- Superior scalability vs. unconstrained HC
- Used in DeepSeek-V4 series

## Limitations (addressed by successors)

- 20 SK iterations ≠ exact doubly stochasticity (approximation gap)
- SK requires specialized CUDA kernels (portability issue)
- O(n³C) parameter complexity in residual stream width n

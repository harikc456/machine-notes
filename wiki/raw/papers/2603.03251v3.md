# Speculative Speculative Decoding

**Authors:** Tanishq Kumar, Tri Dao, Avner May  
**Affiliation:** Stanford University, Princeton University, Together AI  
**Venue:** arXiv preprint, May 2026  
**arXiv:** 2603.03251

## Summary

Introduces Speculative Speculative Decoding (SSD), a framework that eliminates the sequential dependence between drafting and verification in standard speculative decoding. While verification runs on the target model, the draft model simultaneously predicts the likely verification outcome and pre-speculates the next round's tokens for all likely outcomes. SAGUARO is an optimized SSD algorithm.

## Standard SD Sequential Bottleneck

In ordinary speculative decoding:
1. Draft phase: draft model generates γ tokens (sequential)
2. Verify phase: target model verifies all γ tokens in parallel
3. Drafting must wait for verification to complete before starting the next round

This sequential draft→verify→draft→verify chain limits throughput.

## SSD Approach

- Draft model **predicts verification outcome** while verification runs in parallel (on separate hardware)
- Simultaneously **pre-speculates** tokens for all most-likely outcomes
- When actual verification result arrives, draft model checks if it predicted correctly
- If yes: immediately return pre-speculated tokens — **zero drafting overhead**
- Draft model located on distinct hardware from target (e.g., 1×H100 for draft, 4×H100 for target)

## SAGUARO Algorithm

Three optimizations:
1. **Outcome prediction:** Uses most-likely draft logits to predict bonus token (up to 90% accuracy); framed as constrained optimization
2. **Balanced sampling:** Algorithm balances prediction accuracy vs. speculation quality (tension identified and resolved)
3. **Fallback strategy:** Optimal fallback varies with batch size; accounts for failed outcome predictions

## Results (Llama-3.1-70B on TP=4 H100s, Llama-3.2-1B draft)

- **30% faster** than strongest SD baselines (vLLM, SGLang)
- **Up to 5× faster** than autoregressive decoding
- Consistently improves throughput-latency Pareto frontier across batch sizes
- 4.0× end-to-end decode tokens/sec vs. 2.1× (SD) vs. 1.0× (AR)

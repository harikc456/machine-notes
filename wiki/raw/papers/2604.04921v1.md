# TriAttention: Efficient Long Reasoning with Trigonometric KV Compression

**Authors:** Weian Mao, Xi Lin, Wei Huang, Yuxin Xie, Tianfu Fu, Bohan Zhuang, Song Han, Yukang Chen  
**Affiliation:** MIT, NVIDIA, ZJU  
**Venue:** arXiv preprint, April 2026  
**arXiv:** 2604.04921

## Summary

TriAttention compresses KV caches for long-reasoning LLMs by scoring key importance in the **pre-RoPE** space using a trigonometric series built from Q/K concentration centers. This addresses the fundamental instability of post-RoPE importance estimation under long Chain-of-Thought generation.

## Problem with Post-RoPE Methods

Leading KV compression methods (R-KV, H₂O variants) estimate importance using attention scores from recent queries — but RoPE rotation changes query orientations with position. Only the most recent queries retain up-to-date RoPE orientations; too few representative queries → poor key selection and unstable compression under 32K+ reasoning tokens.

## Q/K Concentration Insight

In pre-RoPE space, Q and K vectors cluster tightly around fixed non-zero centers across positions and contexts ("Q/K concentration"). This concentration is:
- Stable across positions (unaffected by positional encoding)
- Directly linked to attention behavior via the RoPE formula
- Measurable via concentration metric R (Mean Resultant Length)

## TriAttention Mechanism

1. Compute Q/K centers (stable pre-RoPE cluster centers per head)
2. Use centers in the RoPE formula → attention logits reduce to a **trigonometric function of Q-K distance** (distance-based attention curve)
3. Score keys by their position relative to the preferred distance peaks
4. For minority heads where Q is less concentrated: fall back to Q/K norms as complementary signal
5. Retain top-scoring keys; evict the rest

## Results (AIME25, 32K-token generation, Qwen3-8B)

- At equivalent accuracy (40.8%): **2.5× higher throughput** than Full Attention
- At matched accuracy: **10.7× KV memory reduction** vs Full Attention
- R-KV achieves only half the accuracy at equivalent efficiency
- Math500: matches Full Attention at 1024/32K tokens retained (68.4% vs 69.6%)
- Enables OpenClaw deployment on a single consumer GPU for long-context reasoning

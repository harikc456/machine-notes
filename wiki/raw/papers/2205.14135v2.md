# FlashAttention: Fast and Memory-Efficient Exact Attention with IO-Awareness

**Authors:** Tri Dao, Daniel Y. Fu, Stefano Ermon, Atri Rudra, Christopher Ré  
**Affiliation:** Stanford University  
**Venue:** NeurIPS 2022  
**arXiv:** 2205.14135

## Summary

FlashAttention reorders the attention computation to minimize HBM (high-bandwidth memory) reads/writes, using on-chip SRAM tiling and online softmax. It computes exact (not approximate) attention while being both faster and more memory-efficient than standard implementations.

## Core Insight

Standard attention is IO-bound: writing/reading the N×N attention matrix to HBM costs O(N²d) bytes of memory bandwidth. FlashAttention avoids materializing this matrix by tiling Q, K, V into blocks that fit in SRAM and accumulating the result using an online softmax update rule.

## Algorithm

Outer loop over Q blocks; inner loop over K/V blocks:
1. Load Q_i, K_j, V_j tiles into SRAM
2. Compute local scores S_ij = Q_i · K_jᵀ / √d
3. Update running softmax statistics (row max m, normalization sum ℓ)
4. Accumulate weighted O_i += softmax_weight · V_j with rescaling
5. Write final O_i to HBM once per Q block

Memory complexity: O(N) (no N×N matrix stored). HBM IO: O(N²d/B) where B is SRAM block size — much less than standard O(N²d).

## Results

- **7.6× faster** than standard PyTorch attention on GPT-2 (A100)
- **1.6× faster** on BERT-large; sets MLPerf training record
- **3× memory reduction** — enables training on 4× longer sequences
- Exact output — no approximation

## Extensions (later work)

- **FlashAttention-2 (2023, 2307.08691):** 2× faster via better work partitioning; fewer non-matmul FLOPs
- **FlashAttention-3 (2024, 2407.08608):** H100-specific; warp specialization + TMA pipeline
- **FlashAttention-4 (2025):** Blackwell GB200; 5-stage pipeline; increased SRAM reuse
- **Flash Decoding:** Parallelizes across sequence length for long-context inference

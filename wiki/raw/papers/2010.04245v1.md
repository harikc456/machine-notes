# Query-Key Normalization for Transformers

**Authors:** Alex Henry, Prudhvi Raj Dachapally, Shubham Pawar, Yuxuan Chen  
**Affiliation:** Cyndx Technologies  
**Venue:** arXiv preprint, October 2020  
**arXiv:** 2010.04245

## Summary

Proposes QKNorm, a simple modification to multi-head attention that ℓ₂-normalizes the query (Q) and key (K) vectors along the head dimension before computing attention scores. This produces cosine-similarity attention:

    Attention(Q, K, V) = softmax( (Q̂ · K̂ᵀ) / τ ) · V

where `Q̂ = Q / ‖Q‖`, `K̂ = K / ‖K‖`, and `τ` is a learned per-head temperature scalar initialized to `√d_k`.

## Key Contributions

- **Prevents softmax saturation:** Standard dot-product attention can produce arbitrarily large logits, driving softmax to near-one-hot distributions. Cosine similarity bounds the logit magnitude, keeping gradients alive.
- **Learned temperature:** The per-head scalar `τ` is learned, allowing the model to control attention sharpness independently.
- **Language modeling improvement:** +0.7 ppl on WikiText-103 over the standard Transformer baseline with no additional parameters beyond `τ`.
- **Consistent gains across tasks:** NMT and LM benchmarks both improved.

## Mechanism

After projecting into query/key spaces but before the scaled dot product, normalize each vector:

    Q̂_{i,h} = Q_{i,h} / ‖Q_{i,h}‖₂
    K̂_{j,h} = K_{j,h} / ‖K_{j,h}‖₂

Dividing by `τ` (not `√d_k`) controls sharpness. The model learns `τ` per head.

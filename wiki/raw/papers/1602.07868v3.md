# Weight Normalization: A Simple Reparameterization to Accelerate Training of Deep Neural Networks

**Authors:** Tim Salimans, Diederik P. Kingma  
**Affiliation:** OpenAI  
**Venue:** NIPS 2016  
**arXiv:** 1602.07868

## Summary

Proposes weight normalization (WN), a reparameterization of neural network weight vectors that decouples the length of weight vectors from their direction:

    w = (g / ‖v‖) · v

where `v` is the weight direction vector and `g` is a scalar magnitude. This is learned via gradient descent jointly with the original parameters. Unlike batch normalization, WN introduces no dependencies between samples in a minibatch, making it well-suited for recurrent models, generative models, and reinforcement learning.

## Key Contributions

- **Reparameterization:** `w = g · v̂` where `v̂ = v / ‖v‖`; gradients flow through both `g` and `v`
- **Mean-only batch normalization:** Lightweight complement — subtracts batch mean but does not divide by variance; reduces noise vs. full BN
- **Data-dependent initialization:** At init, scale `g` and `b` so each neuron's pre-activation has zero mean and unit variance using a single forward pass over a minibatch
- **Conditioning improvement:** WN improves conditioning of the gradient so optimization is faster without mini-batch statistics

## Results

- CIFAR-10: competitive with batch normalization
- MNIST generative model: improved log-likelihood
- Language model: ~21 ppl gain over unnormalized baseline on character-level LM
- Reinforcement learning (Atari): stable training with no batch statistics dependency

## Relationship to Batch Normalization

BN normalizes per-neuron activations across a batch; WN normalizes weight vectors per-neuron independently of batch. WN is deterministic at test time (no running stats), works on any batch size including 1, and composes naturally with recurrent architectures.

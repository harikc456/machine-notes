# SGLang: Efficient Execution of Structured Language Model Programs

**Authors:** Lianmin Zheng, Liangsheng Yin, Zhiqiang Xie, Chuyue Sun, Jeff Huang, Cody Hao Yu, Shiyi Cao, Christos Kozyrakis, Ion Stoica, Joseph E. Gonzalez, Clark Barrett, Ying Sheng  
**Affiliation:** Stanford University, UC Berkeley, SJTU, Texas A&M, Independent  
**Venue:** arXiv preprint (under review), June 2024  
**arXiv:** 2312.07104

## Summary

SGLang is a system for efficient execution of complex "Language Model Programs" (LM programs) — applications that make multiple dependent LLM calls with control flow, structured outputs, and parallel branches. It combines a frontend DSL with a runtime optimized for these multi-call workloads.

## Key Techniques

### RadixAttention (Runtime — KV Cache Reuse)
Maintains an LRU cache of KV tensors across all requests using a **radix tree** indexed by token prefix. When a new request shares a prefix with a cached sequence, the KV cache for that prefix is reused automatically. This is the key cross-request optimization missing from vLLM/TGI. See [[radix-attention]] entity page.

### Compressed Finite State Machines (Runtime — Structured Decoding)
For constrained output (JSON schema, regex), SGLang analyzes the constraint as a finite state machine and compresses multi-token paths into single-step paths when only one valid next-token exists. Decodes multiple tokens in one step — faster than token-by-token masked decoding.

### API Speculative Execution (Runtime — API-only models)
For models accessed via API (GPT-4), SGLang speculatively executes the next LM call before the current one completes, amortizing latency in sequential LM programs.

## Frontend Language

Primitives: `extend`, `gen`, `select` for generation; `fork`, `join` for parallelism. Embedded in Python; compatible with standard Python control flow. Interpreter submits primitives asynchronously to the runtime.

## Results

- **Up to 6.4× higher throughput** vs. state-of-the-art (Guidance, vLLM, LMQL)
- Tested on Llama-7B/70B, Mistral-8x7B, LLaVA-1.5-7B, LLaVA-NeXT-34B; NVIDIA A10G and A100
- Gains driven primarily by RadixAttention on multi-turn and agent workloads

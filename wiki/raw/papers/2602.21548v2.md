# DualPath: Breaking the Storage Bandwidth Bottleneck in Agentic LLM Inference

**Authors**: Yongtong Wu, Shaoyuan Chen, Yinmin Zhong, Rilin Huang, Yixuan Tan, Wentao Zhang, Liyue Zhang, Shangyan Zhou, Yuxuan Liu, Shunfeng Zhou, Mingxing Zhang, Xin Jin, Panpan Huang  
**Affiliation**: Peking University; Tsinghua University; DeepSeek-AI  
**arXiv**: 2602.21548 (Feb 2026)

## Problem

Multi-turn agentic LLM inference is I/O-bound: KV-cache hit rate ≥95%, append length ~429 tokens, cache-compute ratio ~22 GB/PFLOP for DeepSeek-V3.2. In PD-disaggregated systems:
- KV-Cache loads exclusively from storage → prefill-side storage NICs (SNIC) saturate
- Decode-side SNICs idle despite available bandwidth
- GPU compute utilization on prefill engines: 40%; on decode engines: even lower

## Solution: Dual-Path KV-Cache Loading

New storage-to-decode path alongside conventional storage-to-prefill:
1. KV-Cache loads from storage into decode engine memory
2. Transfers from decode engine to prefill engine via RDMA over compute network (CNIC)

Combines this with:
- CNIC-centric traffic management: isolates KV-Cache traffic from model inference collective communications
- Dynamic global scheduler: balances load across prefill/decode engines and both paths

## Key Results

- 1.87× offline inference throughput improvement (vs baseline PD-disaggregated)
- 1.96× online serving throughput improvement on average, without SLO violation
- Evaluated on 3 models with realistic agentic workloads

# EAGLE: Speculative Sampling Requires Rethinking Feature Uncertainty

**Authors:** Yuhui Li, Fangyun Wei, Chao Zhang, Hongyang Zhang  
**Affiliation:** Peking University, Microsoft Research, University of Waterloo, Vector Institute  
**Venue:** arXiv preprint, March 2025  
**arXiv:** 2401.15077

## Summary

EAGLE (Extrapolation Algorithm for Greater Language-model Efficiency) introduces feature-level autoregressive drafting to resolve the "feature uncertainty" problem in speculative sampling. Instead of predicting tokens directly, the draft model autoregressively predicts second-to-top-layer feature vectors of the target LLM, then uses the target LM head to convert features to token distributions.

## Key Observations

1. **Feature autoregression is simpler than token autoregression:** Second-to-top-layer feature sequences are smoother and more regular than token sequences.
2. **Feature uncertainty problem:** In standard speculative sampling, the draft model must predict the next token given the current feature — but sampling introduces uncertainty (multiple tokens could follow `f_t`, leading to different `f_{t+1}`). EAGLE resolves this by inputting the token sequence advanced by one step, making `f_{t+1}` predictable from `(f_t, t_{t+1})`.

## Architecture

- Draft model takes `(f_t, t_{t+1})` as input; predicts `f_{t+1}`
- One auto-regressive FC layer + target LM head
- Trained on 2–4B tokens (target: TinyLLaMA trained on 3000B)
- Draft acceptance rate ~0.8 vs. Medusa's ~0.6 and Lookahead's ~0.5

## Results

- **2.7×–3.5× lossless speedup** on LLaMA2-Chat 70B
- Doubles throughput on LLaMA2-Chat 70B
- Tested across Vicuna, LLaMA2-Chat (7B–70B), Mixtral 8x7B Instruct
- Tasks: dialogue, code generation, math reasoning, instruction following (MT-bench, HumanEval, GSM8K, Alpaca)

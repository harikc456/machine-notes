# EAGLE-3: Scaling up Inference Acceleration of Large Language Models via Training-Time Test

**Authors:** Yuhui Li, Fangyun Wei, Chao Zhang, Hongyang Zhang  
**Affiliation:** Peking University, Microsoft Research, University of Waterloo, Vector Institute  
**Venue:** arXiv preprint, April 2025  
**arXiv:** 2503.01840

## Summary

EAGLE-3 abandons feature prediction entirely in favor of **direct token prediction** and replaces reliance on top-layer features with **multi-layer feature fusion** via a technique called **training-time test**. This removes EAGLE's feature prediction constraint that limited draft model expressiveness and blocked data scaling gains.

## Problems with EAGLE / EAGLE-2

- EAGLE trains on feature prediction loss `l_fea` (predict next feature from current feature)
- Scaling training data yields diminishing returns: feature prediction constraints limit expressiveness
- When feature constraint is removed and training data scales, first draft token acceptance improves — but the predicted feature `â_{t+1}` diverges from ground-truth `f_{t+1}`, causing the second draft token acceptance to collapse (low `1-α`)

## Training-Time Test Technique

Incorporates Step 1 (AR inference on actual model) into training so the draft model trains on realistic inputs: `(f_1, f_2, ..., f_t, â_{t+1})` rather than always using clean ground-truth features. This bridges the train-test distribution gap, enabling multi-step prediction to benefit from more data.

## Architecture Changes

- **Direct token prediction:** Predicts tokens directly, not intermediate features
- **Multi-layer feature fusion:** Uses features from multiple intermediate layers (not just top layer) as conditioning — provides richer signal for multi-step prediction
- Trained with `l_token` only (no `l_fea`)

## Results

- **Up to 6.5× speedup** on LLaMA-Instruct 3.1 8B (MT-bench)
- **~1.4× over EAGLE-2** across tested models
- **1.38× throughput** over EAGLE-2 at batch size 64 in SGLang
- Tested on Vicuna 13B, LLaMA-Instruct 3.1 8B, LLaMA-Instruct 3.3 70B, DeepSeek-R1 LLaMA 8B
- Demonstrates a scaling law: acceptance rate and speedup increase with more training data

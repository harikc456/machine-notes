# Fast Inference from Transformers via Speculative Decoding

**Authors:** Yaniv Leviathan, Matan Kalman, Yossi Matias  
**Affiliation:** Google Research  
**Venue:** ICML 2023  
**arXiv:** 2211.17192

## Summary

Introduces speculative decoding (also called speculative sampling): a lossless inference acceleration algorithm that uses a small, fast "draft" model to propose token candidates, then verifies them in parallel using the large target model. Multiple tokens are produced per target-model forward pass.

## Algorithm

1. **Draft:** Small model `M_q` autoregressively generates `γ` candidate tokens `x̃₁, ..., x̃_γ`
2. **Verify:** Target model `M_p` evaluates all `γ+1` positions in a single forward pass, producing `p(·|context, x̃₁:t)` for each
3. **Accept/Reject:** Token `x̃_t` is accepted with probability `min(1, p(x̃_t)/q(x̃_t))`; if rejected, resample from a corrected distribution and stop
4. **Output:** At least 1 token per round; up to `γ+1` tokens if all accepted

The output distribution is provably identical to sampling from `M_p` alone — no accuracy loss.

## Key Results

- **T5-XXL (11B):** 2–3× wall-clock speedup with T5-Small as draft
- Speedup depends on the draft model's acceptance rate (higher acceptance = more tokens per step)
- Theoretical analysis: expected tokens per round ≥ 1; speedup scales with draft acceptance rate α

## Conditions for Speedup

Speculative decoding is faster when:
- Target model is much larger than draft (latency gap)
- Draft acceptance rate is high (draft and target distributions are similar)
- Batch size is small (target model is memory-bandwidth bound)

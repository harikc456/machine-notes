# Efficient Memory Management for Large Language Model Serving with PagedAttention

**Authors:** Woosuk Kwon, Zhuohan Li, Siyuan Zhuang, Ying Sheng, Lianmin Zheng, Cody Hao Yu, Joseph E. Gonzalez, Hao Zhang, Ion Stoica  
**Affiliation:** UC Berkeley, Stanford University, UC San Diego  
**Venue:** SOSP 2023  
**arXiv:** 2309.06180

## Summary

Proposes PagedAttention, an attention algorithm that divides the KV cache into fixed-size blocks (pages) stored non-contiguously in GPU memory — inspired by OS virtual memory paging. Builds vLLM, an LLM serving system on top of PagedAttention that achieves near-zero KV cache memory waste.

## Problem

Existing systems pre-allocate contiguous KV cache memory per request at maximum sequence length. This causes:
- **Internal fragmentation:** Reserved space unused when actual length < maximum
- **External fragmentation:** Small gaps between allocations too small to use
- Only 20–40% of reserved KV memory holds actual token states in practice

## PagedAttention

- KV cache divided into fixed-size blocks (pages), each holding KV tensors for a fixed number of tokens
- Blocks allocated on demand, stored non-contiguously; a block table maps logical to physical blocks
- **Near-zero waste:** Only the last page per sequence may be partially used
- **Memory sharing:** Requests sharing a common prefix can share physical pages (copy-on-write for divergence); enables efficient beam search and parallel sampling

## vLLM System

Built on PagedAttention:
- Preemptive scheduling: when memory is full, swap or recompute KV for lower-priority sequences
- Supports complex decoding (beam search, parallel sampling) with shared pages
- Compatible with LLaMA, OPT, GPT-2, and other models

## Results

- **2–4× throughput** over FasterTransformer and Orca at the same latency SLO
- Near-zero KV memory waste (96.3% utilization vs. 20–40% in prior systems)
- Improvement scales with sequence length, model size, and complex decoding algorithms

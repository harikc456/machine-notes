# Grouped Query Experts: Mixture-of-Experts on GQA Self-Attention

**Authors**: Vishesh Tripathi, Abhay Kumar  
**Affiliation**: FrontiersMind  
**arXiv**: 2606.20945 (Jun 2026)

## Abstract Summary

Applies MoE conditional routing to the query-head computation within GQA groups, while keeping all KV heads dense. Named GQE (Grouped Query Experts).

**Setup**: Given GQA with G groups, each group has M query-head experts sharing one KV head. A router selects top-k experts per token within the group. One additional always-on shared head per group stabilizes routing. W_O resized from N×d to (kG+2)×d.

**Key distinction from prior work**: routes query heads within fixed GQA groups (not across heads or groups); KV cache is unchanged vs GQA baseline; trains from scratch (no pruning of a dense model).

## Key Results

- 250M parameters, 30B token training budget
- Matches all-active GQA baseline in downstream accuracy while activating 9 of 16 total query-attention heads (including 1 shared head per group)
- 1.7–1.8× prefill speedup at long context (≥32k tokens)
- Speedup grows monotonically with sequence length (attention cost scales quadratically)
